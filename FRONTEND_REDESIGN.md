# 墨·创作 — 前端重构设计方案

## 设计背景

IDE 的加入（文件树 + 编辑器 + AI 侧栏）暴露了当前横向 Tab 导航的问题：
- 8 个 Tab 横向排列拥挤
- IDE 需要更多垂直空间（编辑器纵向滚动体验更好）
- 当前设计隐含假设：少量 Tab + 内容区主导

## 核心改变：横向 → 纵向导航

### 当前架构
```
┌────────────────────────────────────┐
│   TitleBar (42px)                  │
├────────────────────────────────────┤
│ [书库][策划][探讨]...[知识库] (48px)│ ← 横向 Tab
├────────────────────────────────────┤
│                                    │
│   Work Area (内容区)                │
│                                    │
└────────────────────────────────────┘
```

### 新架构（垂直侧栏）
```
┌──┬─────────────────────────────────┐
│  │   TitleBar (36px) ← 更窄       │
├──┼─────────────────────────────────┤
│书│                                 │
│库│                                 │
│  │   Work Area (内容区)            │
│策│   (全高，无横向 Tab 占用)        │
│划│                                 │
│  │                                 │
│探│                                 │
│讨│                                 │
│  │                                 │
│编│                                 │
│辑│                                 │
│  │                                 │
│修│                                 │
│订│                                 │
│  │                                 │
│知│                                 │
│识│                                 │
└──┴─────────────────────────────────┘
  ↑
  64px 窄栏（图标 + 两字标签，垂直排列）
```

## 设计方向：书脊边栏 (Book Spine Sidebar)

**灵感**：古籍书脊——竖排书名，简洁识别，大量书籍并列不显拥挤

**视觉**：
- 左侧窄栏（64px）：每个 Tab 是一个"书脊"
- 图标在上，两字标签竖排在下
- 当前激活：朱砂底色 + 金边
- 未激活：宣纸底 + 墨色图标

**优势**：
- 无限扩展（未来加更多 Tab 不拥挤）
- 给 IDE 留出更多垂直空间
- 文化符号强（书脊＝古籍＝文字创作）
- 独特记忆点（不是通用的左侧宽栏导航）

## 色彩微调（保持水墨体系）

**当前**：宣纸 `#efe8da` + 墨 `#25201c` + 朱砂 `#b73225`

**新增**：
- **金箔**（点缀高亮）：`oklch(76% 0.08 80)` #d4b896
- **墨青**（次级强调）：`oklch(42% 0.03 200)` #5a6c6a

**用途**：
- 金箔：活跃 Tab 的边缘描边
- 墨青：次级按钮、辅助图标

## 排版节奏调整

**标题层级**（保持宋体）：
- H1: 36px / 700 / -0.02em（屏幕标题）
- H2: 28px / 600（区块标题）
- H3: 22px / 600（子区块）
- Body: 15px / 400 / 1.75（正文）

**行高优化**（IDE 需要更紧凑）：
- 创作/阅读屏：1.75（现有）
- IDE 编辑器内：1.6（CodeMirror 内部已设 1.9，这里指 UI 层）
- 列表/菜单：1.5

## 空间韵律

**当前**：统一 `--sp-*` 刻度

**优化**：
- **创作屏**（宽松）：`--sp-6`(32px) panel padding
- **IDE 屏**（紧凑）：`--sp-4`(18px) panel padding
- **导航栏**（精简）：`--sp-2`(8px) item gap

## 组件重构清单

### 1. AppShell（app.css）
- 改为 `grid-template-columns: 64px 1fr`
- 去掉 `<WorkTabs>` 的横向占位
- TitleBar 高度 42px → 36px（更紧凑）

### 2. Navigation（新建 sidebar.css）
- 垂直书脊式导航条
- 每项：图标 18px + 竖排两字标签
- 激活态：朱砂底 + 金边 + 微凸起阴影
- Hover：墨洗浮起

### 3. IDE（ide.css 微调）
- 去掉顶部 Tab 栏的 36px 浪费（已有侧栏导航）
- 文件树宽度 220px → 200px（侧栏占了空间，稍微收窄）
- AI 面板 300px → 280px（同理）

### 4. 各工作屏（work-screens.css）
- 去掉内嵌的二级 Tab（所有导航统一到侧栏）
- 利用多出的垂直空间

## 动效细节

**侧栏 Tab 切换**：
- 激活态朱砂底色：ease-out-quint 280ms
- 金边从 0 → 2px：ease-out-quart 180ms
- 微凸起：`box-shadow` 渐变 220ms

**屏幕切换**：
- 当前屏淡出（opacity 0, transform: translateX(-8px)）120ms
- 新屏淡入（opacity 0→1, transform: translateX(8px)→0）280ms ease-out-quint

## 响应式策略

**窄屏（< 900px）**：
- 侧栏收起为纯图标（40px 宽）
- Hover 展开浮动 tooltip 显示文字
- IDE 的 AI 面板默认收起（底部浮动按钮呼出）

**超宽屏（> 1600px）**：
- IDE 的编辑器 max-width: 960px 居中（长行文字不利阅读）
- 两侧留白增加，不无限拉宽

## 实施步骤

1. **设计 Token 更新**（theme.css）
   - 新增金箔、墨青色值
   - 调整间距刻度

2. **AppShell 重构**（app.css）
   - Grid 布局改为两栏
   - TitleBar 瘦身

3. **新建 Sidebar 导航**（navigation.css + Navigation.tsx）
   - 垂直书脊式
   - 替换 WorkTabs

4. **IDE 适配**（ide.css）
   - 去掉冗余高度
   - 宽度微调

5. **各屏优化**（逐屏调整 padding / 去二级 Tab）

6. **动效抛光**（transition 统一）

---

**预期效果**：
- 垂直空间利用率 ↑ 12%（去掉 48px 横向 Tab）
- 导航可扩展性 ↑ 无限（书脊纵向叠加）
- 文化识别度 ↑ "书脊"隐喻强化"文字工具"定位
- IDE 体验 ↑ 更接近专业编辑器的纵向主导布局
